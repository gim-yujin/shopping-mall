# 결정기록: Tier 정책 (2026-03-14)

대상 독자: 운영자

## 결정 사항
- **확정 정책:** 회원 등급 정기 점검은 `users.total_spent`(누적 구매금액) 기준으로 산정한다.
- **비적용 정책:** 매년 전년도 실적으로 `total_spent`를 리셋하는 연간 리셋 방식은 채택하지 않는다.

## 근거

### 1) 비즈니스 의미
- 누적 기준은 장기 고객의 기여도를 일관되게 반영한다.
- 일시적 비수기/시즌성으로 인한 급격한 등급 하락을 줄여 고객 충성도 프로그램의 예측 가능성을 높인다.

### 2) 고객 커뮤니케이션
- “지금까지 누적 구매금액 기반 등급”은 고객이 직관적으로 이해하기 쉽고, 마이페이지에서 안내하기 용이하다.
- 연초 리셋 정책 대비 문의/불만(“작년에 많이 샀는데 왜 초기화됐나요?”) 가능성을 낮춘다.

### 3) 회계/운영 관점
- 전년도 실적은 회계/분석 리포트 지표로 분리 관리하고, 혜택 산정 지표(누적 total_spent)와 역할을 분리한다.
- 정책/리포트 지표가 분리되면 등급 혜택 변경과 회계 보고 변경을 독립적으로 운영할 수 있어 변경 영향 범위가 줄어든다.

## 구현 반영 포인트
- `TierScheduler`는 전년도 주문금액을 집계하되, 등급 판정에는 누적 구매금액을 사용한다.
- 전년도 금액은 등급 변경 이력 문구/리포팅 참고 데이터로만 사용한다.
