# 운영가이드: 문서 수치 갱신 규칙

대상 독자: 운영자

## 목적
프로젝트 문서(보고서/README/위키)에 포함되는 DB 구조 수치(테이블 수, 인덱스 수)가 실제 스키마와 달라지는 드리프트를 방지한다.

## 단일 출처(SSOT)
- 수치의 **유일한 출처는 `src/main/resources/schema.sql`** 이다.
- 문서의 임의 추정치(예: "약 50개", "15개")는 금지하고, 재집계한 실제 값을 사용한다.

## 집계 기준
- 테이블 수: `CREATE TABLE` 구문 개수
- 인덱스 수: `CREATE INDEX` + `CREATE UNIQUE INDEX` 구문 개수
- 고유 인덱스 수: `CREATE UNIQUE INDEX` 구문 개수
- 일반 인덱스 수: `CREATE INDEX` 구문 개수(단, `CREATE UNIQUE INDEX`는 별도 집계)

> 주의: `--` 주석에 포함된 문자열은 집계에서 제외한다.

## 표준 재집계 절차
아래 명령으로 문서 수치(코드/템플릿/스키마)를 한 번에 검증한다.

```bash
./scripts/validate-doc-stats.sh
```

- 결과는 Markdown 표로 출력된다.
- `❌ 드리프트`가 하나라도 있으면 종료 코드 1로 실패한다.

스키마 세부 수치만 단독 확인할 때는 아래 명령을 사용할 수 있다.

```bash
python - <<'PY'
import re, pathlib
p = pathlib.Path('src/main/resources/schema.sql')
text = p.read_text(encoding='utf-8')
clean = '\n'.join(line for line in text.splitlines() if not re.match(r'^\s*--', line))

count_tables = len(re.findall(r'\bCREATE\s+TABLE\b', clean, flags=re.I))
count_unique = len(re.findall(r'\bCREATE\s+UNIQUE\s+INDEX\b', clean, flags=re.I))
count_index_all = len(re.findall(r'\bCREATE\s+(?:UNIQUE\s+)?INDEX\b', clean, flags=re.I))
count_index_normal = count_index_all - count_unique

print('tables=', count_tables)
print('indexes_total=', count_index_all)
print('indexes_normal=', count_index_normal)
print('indexes_unique=', count_unique)
PY
```

## 문서 반영 규칙
1. `schema.sql` 변경 PR에서는 문서 수치를 같은 PR에서 함께 갱신한다.
2. 다음 위치를 최소 점검 대상으로 한다.
   - `ShopMall_프로젝트_보고서_0.md` 2장(데이터베이스 설계)
   - `ShopMall_프로젝트_보고서_0.md` 8장(프로젝트 구조 요약)
   - 그 외 DB 구조 수치가 등장하는 README/문서
3. 표기 권장안
   - 예: `17개 테이블, 52개 인덱스(일반 50개 + UNIQUE 2개)`
4. 수치 변경 시, 커밋 메시지 또는 PR 본문에 재집계 명령과 결과를 함께 남긴다.
